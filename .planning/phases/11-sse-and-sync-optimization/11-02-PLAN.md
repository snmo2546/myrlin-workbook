  <name>Task 2: Add workspace-aware SSE filtering to broadcastSSE</name>

Modify the `broadcastSSE` function to filter events by workspace subscriptions:

1. **Load subscriptions into SSE client on connect**: In the `/api/events` handler, after storing the client in sseClients, if `deviceId` is present, look up the device and load its `workspaceSubscriptions` into `client.subscriptions`:
   ```javascript
   if (deviceId) {
     const device = getStore().findDevice(deviceId);
     client.subscriptions = device?.workspaceSubscriptions || null;
   }
   ```
   A `null` value means "no filtering" (receive all). An empty array `[]` also means "receive all" per WSUB-04.

2. **Filter in broadcastSSE**: Modify the broadcast loop to skip workspace-scoped events for clients with active subscriptions:
   ```javascript
   function broadcastSSE(eventType, data) {
     const payload = JSON.stringify({ type: eventType, data, timestamp: new Date().toISOString() });
     const message = `data: ${payload}\n\n`;

     // Extract workspaceId from event data (various shapes)
     const workspaceId = data?.workspaceId || data?.workspace?.id || data?.id || null;
     const isGlobal = GLOBAL_EVENT_TYPES.has(eventType);

     for (const [clientId, client] of sseClients) {
       if (client.res.writableEnded) {
         clearInterval(client.heartbeatInterval);
         sseClients.delete(clientId);
         continue;
       }

       // Subscription filtering: only applies to non-global, workspace-scoped events
       // when the client has an active subscription list with entries
       if (!isGlobal && workspaceId && client.subscriptions && client.subscriptions.length > 0) {
         if (!client.subscriptions.includes(workspaceId)) continue;
       }

       try {
         client.res.write(message);
       } catch (_) {
         clearInterval(client.heartbeatInterval);
         sseClients.delete(clientId);
       }
     }
   }
   ```

   Key rules:
   - `GLOBAL_EVENT_TYPES` events always sent (SSE-05)
   - If client has no subscriptions (null or empty array), send everything (WSUB-04)
   - If client has subscriptions and event has a workspaceId not in the list, skip
   - If event has no workspaceId (e.g., template events), send to all (safe default)
   - Desktop browser clients (no deviceId) never have subscriptions, so they get everything

3. **Handle workspace:created/updated/deleted events**: These events have the workspace `id` directly in data, not `workspaceId`. The extraction logic above handles this via `data?.id`. But be careful: for `workspace:deleted`, the event data is `{ id, name }` and should be sent to clients subscribed to that workspace ID so they can remove it. For `workspace:created`, it should go to all clients (it is a new workspace, nobody is subscribed yet). Add `workspace:created` to `GLOBAL_EVENT_TYPES`.

4. **Also clear heartbeatInterval** in any catch blocks where clients are removed (already done if 11-01 stored it on the record). Ensure consistency.
