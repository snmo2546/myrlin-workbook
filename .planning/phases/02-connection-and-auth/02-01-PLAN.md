  <action>
1. **Modify src/web/auth.js** to export `generateToken` and `addToken` (a new function that wraps `activeTokens.add(token)`). Add these to `module.exports`. Do NOT remove any existing exports. The `addToken` function simply calls `activeTokens.add(token)` so pairing.js can register new tokens without directly accessing the Set.

2. **Create src/web/pairing.js** implementing the pairing flow from the research doc:
   - `const PAIRING_TTL_MS = 5 * 60 * 1000` (5 minutes)
   - `const pairingTokens = new Map()` for in-memory pairing token storage
   - Cleanup interval every 2 minutes to delete expired/used tokens (call `.unref()`)
   - `setupPairing(app, { requireAuth, addToken, generateToken, isRateLimited })` function:
     - **GET /api/auth/pairing-code** (requireAuth protected):
       - Generates 32-byte random hex pairing token
       - Stores in pairingTokens Map with `{ createdAt: Date.now(), used: false }`
       - Reads server URL from `req.query.url` or falls back to `req.protocol + '://' + req.get('host')`
       - Returns `{ pairingToken, expiresAt, qrPayload }` where qrPayload is JSON string with `{ url, pairingToken, serverName, version }` from package.json
     - **POST /api/auth/pair** (public, rate-limited):
       - Validates `pairingToken` field exists and is a string
       - Checks pairingTokens Map: rejects if not found, already used, or expired
       - Deletes the pairing token after successful use (single-use)
       - Generates a long-lived Bearer token via `generateToken()` and registers via `addToken(token)`
       - Returns `{ success: true, token, serverName, serverVersion }`
       - Rate limited via `isRateLimited(clientIp)`, returns 429 on limit
   - `module.exports = { setupPairing }`

3. **Modify src/web/server.js** to mount pairing routes:
   - Add `const { setupPairing } = require('./pairing');` near the top imports
   - Import `isRateLimited` from auth.js (add to existing destructure, or re-export it from auth.js)
   - Call `setupPairing(app, { requireAuth, addToken: require('./auth').addToken, generateToken: require('./auth').generateToken, isRateLimited: require('./auth').isRateLimited })` right after `setupAuth(app)` on line 272
   - Note: `isRateLimited` is currently not exported from auth.js. Add it to the module.exports.

4. **Create test/pairing.test.js** with integration tests:
   - Test GET /api/auth/pairing-code requires auth (401 without token)
   - Test GET /api/auth/pairing-code returns valid pairing token and QR payload with auth
   - Test POST /api/auth/pair with valid pairing token returns success + Bearer token
   - Test POST /api/auth/pair with invalid token returns 403
   - Test POST /api/auth/pair with expired token returns 403
   - Test POST /api/auth/pair used token returns 403 (single-use)
   - Test rate limiting on POST /api/auth/pair
   - Follow existing test patterns from test/ directory (use the server's existing test setup)

2. **Create mobile/types/api.ts** with shared types:
   - `ServerConnection` interface: `{ id: string, name: string, url: string, token: string, pairedAt: string, lastConnected: string, tunnelType: 'lan' | 'cloudflare' | 'tailscale' | 'relay' }`
   - `ConnectionStatus` type: `'connected' | 'disconnected' | 'connecting' | 'token-expired'`
   - `PairingCodeResponse`: `{ pairingToken: string, expiresAt: string, qrPayload: string }`
   - `PairResponse`: `{ success: boolean, token: string, serverName: string, serverVersion: string }`
   - `LoginResponse`: `{ success: true, token: string } | { success: false, error: string }`
   - `AuthCheckResponse`: `{ authenticated: boolean }`
   - `APIError` class extending Error with `status: number` and `body: string` fields

3. **Create mobile/stores/secure-storage-adapter.ts**:
   - Import `* as SecureStore` from 'expo-secure-store'
   - Import `createJSONStorage` and `StateStorage` type from 'zustand/middleware'
   - Implement `secureStorage: StateStorage` with async `getItem` (SecureStore.getItemAsync), `setItem` (SecureStore.setItemAsync), `removeItem` (SecureStore.deleteItemAsync)
   - Export `createSecureStorage` function that returns `createJSONStorage(() => secureStorage)`
   - Add JSDoc comments on every function

4. **Create mobile/stores/server-store.ts**:
   - Import types from '../types/api'
   - Follow the Zustand persist pattern from theme-store.ts but use `createSecureStorage()` instead of MMKV
   - State: `servers: ServerConnection[]`, `activeServerId: string | null`, `connectionStatus: ConnectionStatus`, `_hasHydrated: boolean`
   - Derived getter: `activeServer` computed from servers + activeServerId
   - Actions:
     - `addServer(data: Omit<ServerConnection, 'id' | 'pairedAt' | 'lastConnected'>)`: generates UUID, sets as active, sets connectionStatus to 'connected'
     - `removeServer(id)`: removes server, falls back activeServerId to first remaining server or null
     - `switchServer(id)`: sets activeServerId, sets connectionStatus to 'connecting'
     - `setConnectionStatus(status)`: updates connectionStatus
     - `updateToken(serverId, token)`: updates token for a specific server
     - `logout(serverId)`: clears token for that server, sets connectionStatus to 'disconnected' if it was the active server
     - `setHasHydrated(val: boolean)`: sets _hasHydrated flag
   - Persist config: name 'myrlin-servers', storage: createSecureStorage(), partialize to only persist `servers` and `activeServerId`
   - Use `onRehydrateStorage` to set `_hasHydrated = true` when hydration completes

5. **Create mobile/services/api-client.ts**:
   - `MyrlinAPIClient` class with constructor `(baseUrl: string, token: string)`
   - Private `_fetch(path, options?)` method that:
     - Prepends baseUrl to path
     - Adds `Authorization: Bearer {token}` header
     - Adds `Content-Type: application/json` for POST/PUT/PATCH
     - Throws `APIError` on non-ok responses
   - Methods (typed return values from api.ts):
     - `checkAuth(): Promise<AuthCheckResponse>` - GET /api/auth/check
     - `login(password: string): Promise<LoginResponse>` - POST /api/auth/login
     - `logout(): Promise<{ success: boolean }>` - POST /api/auth/logout
     - `pair(pairingToken: string, deviceName: string, platform: string): Promise<PairResponse>` - POST /api/auth/pair (no auth header needed, omit Bearer for this call)
   - Export `createAPIClient(baseUrl: string, token: string): MyrlinAPIClient`
   - Add JSDoc comments on class and every method
