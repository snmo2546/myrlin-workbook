  - `returnKeyType="send"`, `blurOnSubmit={false}` (keep keyboard open after send)
  - `autoCapitalize="none"`, `autoCorrect={false}`, `spellCheck={false}` (terminal input)
  - `style`: bgElevated background, textPrimary color, fontMono, fontSize sm (13), rounded corners, flex: 1

**2. mobile/components/terminal/TerminalToolbar.tsx**:
- Props: `sendToWebView: (msg: ToWebView) => void`, `onText: (text: string) => void` (callback when text is received from bridge for clipboard/share operations)
- Toolbar buttons (horizontal ScrollView, horizontal padding md):
  - **Copy**: Ionicons "copy-outline". On press: `sendToWebView({ type: 'getVisibleText' })`. The bridge's onText callback will receive the text; use `Clipboard.setStringAsync(text)` to copy it. Show brief "Copied!" feedback.
  - **Paste**: Ionicons "clipboard-outline". On press: read clipboard via `Clipboard.getStringAsync()`, then `sendToWebView({ type: 'write', data: clipboardText })` to inject into PTY.
  - **Share**: Ionicons "share-outline". On press: `sendToWebView({ type: 'getVisibleText' })`. On text received, call `Share.share({ message: text })` from react-native's Share API.
  - **Camera**: Ionicons "camera-outline". On press: launch `ImagePicker.launchImageLibraryAsync({ mediaTypes: ['images'], quality: 0.8 })`. If not canceled, show an alert or prepare for upload. Note: actual upload endpoint may need a server-side route. For now, convert image URI to a brief status message. If no upload endpoint exists, show an Alert explaining the image was selected but upload requires server support.
  - **Reader**: Ionicons "book-outline". On press: calls a `onReaderToggle` callback prop (implemented in plan 03).
- Each button: 40x40 pressable, centered icon (size 20), textSecondary color, bgElevated on press (opacity feedback)
- Separator line (1px borderSubtle) between toolbar and input

Import `expo-clipboard` as `* as Clipboard from 'expo-clipboard'`, `Share` from 'react-native', `* as ImagePicker from 'expo-image-picker'`.

**Voice input (TERM-07):** iOS natively supports dictation via the microphone button on the keyboard. Since TerminalInput uses a standard TextInput, dictation works automatically. **KeyboardStickyView** from 'react-native-keyboard-controller':
- Import `KeyboardProvider` and `KeyboardStickyView`
- Wrap the entire screen content in `KeyboardProvider` (if not already at app root). Check if it can be added to the app root _layout.tsx instead. If so, add it there. If adding at root is risky mid-phase, wrap locally in TerminalScreen.
- `KeyboardStickyView` wraps the toolbar + input so they stick above the keyboard
- The WebView area should shrink when the keyboard opens. KeyboardStickyView handles this automatically by pushing content up.

**Wire the input to the bridge:**
- TerminalInput `onSubmit` -> `sendToWebView({ type: 'write', data: text })`. The text already includes '\n' from TerminalInput.
- TerminalToolbar receives `sendToWebView` and `onText` callback
- The bridge hook's `onText` callback routes text responses (selectedText, visibleText, scrollback) to a state variable or ref that the toolbar can consume

**Wire text responses for clipboard/share:**
- Add state: `pendingTextAction: 'copy' | 'share' | null`
- When toolbar Copy is pressed: set pendingTextAction to 'copy', send getVisibleText
- When toolbar Share is pressed: set pendingTextAction to 'share', send getVisibleText
- When bridge onText fires with visibleText: check pendingTextAction, execute copy or share, clear pendingTextAction
- This two-step pattern is necessary because the bridge is async (postMessage round-trip)

**Resize terminal on keyboard show/hide:**
- Import `useKeyboardHandler` from react-native-keyboard-controller (or use KeyboardStickyView's built-in behavior)
- When keyboard height changes significantly, optionally send a resize to the WebView. However, since KeyboardStickyView handles the layout shift, the WebView's ResizeObserver (in terminal.html) should detect the size change and re-fit automatically via fitAddon.fit(). Test this; if the WebView doesn't resize, add an explicit resize message.

**Reader mode toggle placeholder:**
- Pass `onReaderToggle` prop to TerminalToolbar that sets a state boolean
- When reader mode is true, show a placeholder View with "Reader mode coming in plan 03" text
- This ensures the prop chain is wired for plan 03

**Ensure KeyboardProvider is in the component tree:**
- The safest place is wrapping TerminalScreen's return. Add `<KeyboardProvider>` around the outermost View. This avoids modifying the app root layout.
