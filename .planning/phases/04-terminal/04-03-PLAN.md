**1. mobile/components/terminal/ReaderMode.tsx**:

**2. mobile/components/terminal/TerminalCarousel.tsx**:
- Props: `sessionIds: string[]`, `initialIndex: number`, `onPageChange?: (index: number) => void`
- Install react-native-pager-view: `npx expo install react-native-pager-view`
- Use PagerView from react-native-pager-view for native page swiping
- **Lazy WebView mounting (CRITICAL for memory):** Only render the full TerminalScreen (with WebView) for the currently active page. For non-active pages, render a placeholder view with the session name and "Swipe to activate" text on the theme background.
- Track `activeIndex` via `onPageSelected` callback from PagerView
- When activeIndex changes:
  1. The previously active terminal sends `dispose` to clean up its WebView/WebSocket
  2. The newly active terminal mounts its WebView and connects
  3. Call `onPageChange?.(newIndex)` for parent to update header
- Page indicator dots at the bottom (small circles, accent for active, surface1 for inactive). Only show if more than 1 session.
- If only 1 session, render TerminalScreen directly without PagerView wrapper

**Implementation notes:**
- PagerView provides native iOS/Android page swiping with correct physics and deceleration
- Each page is a View with `key={sessionId}` to ensure correct recycling
- The "only mount active WebView" pattern prevents memory issues from multiple xterm.js instances

- Add `readerMode` state boolean (default false)

- When readerMode is toggled ON:

  2. Store received scrollback text in state
  3. Render `<ReaderMode text={scrollbackText} onClose={() => setReaderMode(false)} isLoading={!scrollbackText} />`

- Accept optional `isActive` prop (default true). When `isActive` is false (carousel non-active page), do NOT mount the WebView. Render a placeholder instead: session name centered on base background with a terminal icon. This is how TerminalCarousel controls lazy mounting.
