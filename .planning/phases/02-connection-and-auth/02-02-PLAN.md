   - Welcome screen shown on first launch (no servers paired)
   - Myrlin logo/icon at top (use Ionicons 'code-working-outline' as placeholder)
   - "Welcome to Myrlin" title, brief description text
   - Two primary action buttons using theme colors:
     - "Scan QR Code" (navigates to scan-qr, icon: 'qr-code-outline')
     - "Connect Manually" (navigates to manual-connect, icon: 'link-outline')
   - Use `useTheme()` hook for all colors, spacing, radius
   - Style: centered layout, Catppuccin themed, Plus Jakarta Sans font

3. **Create mobile/app/(auth)/scan-qr.tsx**:
   - Import `CameraView` and `useCameraPermissions` from 'expo-camera'
   - Check camera permissions first. If not granted, show a permission request UI with a "Grant Camera Permission" button and a fallback "Connect Manually" link
   - CameraView fills screen with `barcodeScannerSettings={{ barcodeTypes: ['qr'] }}`
   - `onBarcodeScanned` handler with `scanned` state flag to prevent multiple fires:
     - Parse QR data as JSON: extract `{ url, pairingToken, serverName, version }`
     - Call `createAPIClient(url, '').pair(pairingToken, deviceName, platform)` (no auth needed for pair)
     - On success: call `useServerStore.getState().addServer({ name: serverName, url, token: result.token, tunnelType: 'lan' })`
     - Navigate to `/(tabs)/sessions` via `router.replace('/(tabs)/sessions')`
     - On error: show error message, set scanned=false to allow retry
   - Overlay: semi-transparent dark overlay with a transparent square "viewfinder" in the center
   - Back button in top-left corner
   - "Scan the QR code shown in Myrlin desktop" instructional text below viewfinder
   - Get device name from expo-device if available, fallback to 'Mobile Device'
   - Get platform from `Platform.OS`

4. **Create mobile/app/(auth)/manual-connect.tsx**:
   - Form with three fields:
     - Server URL (TextInput, placeholder "http://192.168.1.50:3456", autoCapitalize: none, keyboardType: url)
     - Password (TextInput, secureTextEntry: true)
     - Server Name (TextInput, optional, placeholder "My Server")
   - "Connect" button (disabled while loading, shows activity indicator)
   - URL validation: strip trailing slashes, default to http:// if no protocol, validate with URL constructor. Show inline error for invalid URLs.
   - On submit:
     - Normalize URL
     - Call `createAPIClient(url, '').login(password)` (no auth header for login)
     - On success: `addServer({ name: serverName || 'Myrlin Server', url, token: result.token, tunnelType: 'lan' })`
     - Navigate to `/(tabs)/sessions`
     - On error: show error message below form (red text, theme.colors.error)
   - Back button navigates to onboarding
   - All styled with useTheme(), proper keyboard avoidance (KeyboardAvoidingView)

5. **Create mobile/app/(auth)/login.tsx**:
   - Used when a known server's token has expired (passed via route params: `serverId`, `serverUrl`, `serverName`)
   - Shows server name at top
   - Password field and "Log In" button
   - On submit: call login, then `updateToken(serverId, newToken)` and `setConnectionStatus('connected')`
   - Navigate back to previous screen on success
   - "Scan QR Instead" link at bottom for re-pairing
   - Handle error states with clear messages
