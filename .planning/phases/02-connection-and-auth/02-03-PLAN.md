<task type="auto">
  <name>Task 1: Auth store, biometric gate, and connection health hook</name>
  <files>mobile/stores/auth-store.ts, mobile/components/BiometricGate.tsx, mobile/hooks/useConnectionHealth.ts, mobile/components/ConnectionDot.tsx</files>
  <action>
1. **Create mobile/stores/auth-store.ts**:
   - Zustand store with MMKV persistence (same adapter pattern as theme-store.ts)
   - State: `biometricEnabled: boolean` (user preference), `isLocked: boolean` (current lock state)
   - Actions:
     - `setBiometricEnabled(enabled: boolean)`: toggles biometric lock preference. When enabling, call `LocalAuthentication.hasHardwareAsync()` and `isEnrolledAsync()` first. If no hardware or not enrolled, do not enable (return false or throw).
     - `lock()`: sets isLocked=true (call on app background)
     - `unlock()`: sets isLocked=false (call after successful biometric auth)
   - Persist only `biometricEnabled` to MMKV (not `isLocked`, which starts as false on launch)
   - Use the existing MMKV pattern from theme-store.ts with a dedicated `createMMKV({ id: 'myrlin-auth' })` instance

2. **Create mobile/components/BiometricGate.tsx**:
   - Full-screen overlay (StyleSheet.absoluteFillObject, zIndex 9999)
   - Background: theme.colors.base
   - Centered content: lock icon (Ionicons 'lock-closed-outline'), "Myrlin is locked" text, "Tap to unlock" pressable button
   - On mount (useEffect), automatically call `LocalAuthentication.authenticateAsync({ promptMessage: 'Unlock Myrlin', fallbackLabel: 'Use Passcode', disableDeviceFallback: false })`
   - On success: call `useAuthStore.getState().unlock()`
   - On failure: show the "Tap to unlock" button for manual retry
   - Use Reanimated for fade-in animation (opacity from 0 to 1, 200ms)
   - All colors from useTheme()

3. **Create mobile/hooks/useConnectionHealth.ts**:
   - Takes no arguments, reads activeServer from useServerStore
   - useEffect that runs when `activeServer?.id` changes:
     - Creates a polling loop with exponential backoff
     - Initial interval: 5000ms
     - On success (authenticated=true): set connectionStatus 'connected', reset backoff to 5000ms
     - On success (authenticated=false): set connectionStatus 'token-expired'
     - On network error: set connectionStatus 'disconnected', double backoff (max 60000ms)
     - Cleanup: clearTimeout on unmount or server change
   - Also listens to AppState changes (from 'react-native'): when app comes to foreground, immediately check health (do not wait for next interval)
   - When app goes to background and biometricEnabled is true, call `useAuthStore.getState().lock()`
   - Export `useConnectionHealth` hook

4. **Create mobile/components/ConnectionDot.tsx**:
   - Small colored dot component (8x8, borderRadius full)
   - Props: none (reads connectionStatus from useServerStore directly)
   - Colors from theme:
     - 'connected' = theme.colors.green
     - 'connecting' = theme.colors.yellow
     - 'disconnected' = theme.colors.red
     - 'token-expired' = theme.colors.peach
   - Animated pulse for 'connecting' state (Reanimated opacity animation, repeating)
   - Optional `showLabel` prop that shows text label next to dot
   - Add JSDoc comments
  </action>
